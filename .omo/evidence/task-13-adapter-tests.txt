Key decisions
-------------
1. user_id column must be provided as "" (not omitted) in the NullTolerant fixture
   because hermesSession.UserID is a plain string and database/sql cannot scan NULL
   into a non-pointer string type.
2. NullTolerant test uses a separate hand-built DB rather than the shared fixture
   so the NULL condition can be injected cleanly without modifying NewFixtureDB.
3. Detect() correctly filters fixture-child-001 (has ParentSessionID) — only 2 root
   sessions returned: fixture-parent-001 and cron_job123_1700000000.
